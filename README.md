# Mermaid Forge

Native Windows 11 desktop app that converts Mermaid diagrams to editable
PowerPoint (.pptx) and Visio (.vsdx) files. Built on Tauri 2.x.

## Prerequisites (Windows 11)

Install once per machine.

- Rust 1.94+ via rustup (`winget install Rustlang.Rustup`)
- Tauri CLI 2.x (`cargo install tauri-cli --version "^2.0" --locked`)
- Node.js 22 LTS (`winget install OpenJS.NodeJS.LTS`)
- Python 3.13 via miniconda at `E:\miniconda3\` (the build-sidecar script
  has this path hard-coded; edit if your Python is elsewhere)
- Python packages: `pip install python-pptx pyinstaller`
- Graphviz: `winget install Graphviz.Graphviz` (add to PATH if installer
  doesn't: `C:\Program Files\Graphviz\bin`)
- WebView2 Runtime (preinstalled on Win11)

## Build from a fresh clone

```powershell
cd <path-to-mermaidforge>
cd app
npm install      # one-time per fresh clone
cd ..
npm --prefix app run build
```

The `npm run build` chain:
1. `build-sidecar.ps1` copies engines from `engines/` into the sidecar
   working folder, runs PyInstaller, verifies, and installs the result
   into `app/src-tauri/binaries/`.
2. `tauri build` produces the `.msi` installer.

Output: `app/src-tauri/target/release/bundle/msi/Mermaid Forge_*.msi`

## Quick rebuild during development

If you only changed Rust/JS code (no engine changes):
```powershell
npm --prefix app run build:app
```

If you only changed engine Python code:
```powershell
.\build-sidecar.ps1
```

## Project layout

```
mermaidforge/
├── app/                       # Tauri 2.x application
│   ├── src/                   # Frontend (vanilla JS + Mermaid.js)
│   ├── src-tauri/             # Rust backend + sidecar config
│   │   ├── python/
│   │   │   ├── cli.py         # Sidecar entrypoint (PIL stub)
│   │   │   └── mfengine.spec  # PyInstaller config
│   │   └── binaries/          # Frozen sidecar (gitignored, regenerable)
│   └── package.json
├── engines/                   # Python engines (SOURCE OF TRUTH)
│   ├── mermaid_to_pptx.py
│   ├── mermaid_to_vsdx_generator.py
│   └── template_manual_ref.vsdx
├── design/                    # Icons, design refs
├── docs/test-corpus/          # Mermaid sample files shipped as templates
└── build-sidecar.ps1          # Engine → sidecar build script
```

Engine files are stored canonically in `engines/`. The sidecar working
copies under `app/src-tauri/python/` are regenerated by
`build-sidecar.ps1` and are not tracked in git.

## Updating the app icon

The app icon source is `design/icon-master.png` (currently 2048×2048).
After changing it, regenerate the platform icon set:

```powershell
cd app
npx tauri icon ..\design\icon-master.png
```

This populates `app/src-tauri/icons/` with all sized variants Tauri
needs (32x32, 128x128, 128x128@2x, icon.ico, icon.icns, and various
MS Store / iOS / Android variants — the unused mobile assets stay on
disk but don't ship in the Win11 MSI).

The 24×24 toolbar icon at `app/src/icon-24.png` is generated separately
(Tauri's `icon` command doesn't emit a 24×24). Regenerate with this
PowerShell snippet:

```powershell
Add-Type -AssemblyName System.Drawing
$src = "design\icon-master.png"
$dst = "app\src\icon-24.png"
$srcImg = [System.Drawing.Image]::FromFile((Resolve-Path $src).Path)
$dstImg = New-Object System.Drawing.Bitmap(24, 24)
$g = [System.Drawing.Graphics]::FromImage($dstImg)
$g.InterpolationMode = [System.Drawing.Drawing2D.InterpolationMode]::HighQualityBicubic
$g.SmoothingMode = [System.Drawing.Drawing2D.SmoothingMode]::HighQuality
$g.PixelOffsetMode = [System.Drawing.Drawing2D.PixelOffsetMode]::HighQuality
$g.DrawImage($srcImg, 0, 0, 24, 24)
$dstImg.Save((Resolve-Path $dst).Path, [System.Drawing.Imaging.ImageFormat]::Png)
$g.Dispose(); $dstImg.Dispose(); $srcImg.Dispose()
```

Rebuild with `npm --prefix app run build:app` to pick up icon changes
in the MSI.

## Known limitations

- VSDX export does not currently handle cyclic diagrams (engine layout
  layer). The app detects cycles before export and shows a clear
  message. PPTX export handles cycles fine.
- Native export is only available for `flowchart` diagrams. Other
  Mermaid types (sequence, class, ER, etc.) render in the preview but
  do not export to native shapes.
