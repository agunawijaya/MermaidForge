# Building Mermaid Forge from source

Targets Windows 11. The build produces an `.msi` installer in
`app/src-tauri/target/release/bundle/msi/`.

## Prerequisites

Install once per machine.

- **Rust 1.94+** via rustup — `winget install Rustlang.Rustup`
- **Tauri CLI 2.x** — `cargo install tauri-cli --version "^2.0" --locked`
- **Node.js 22 LTS** — `winget install OpenJS.NodeJS.LTS`
- **Python 3.13** via miniconda at `E:\miniconda3\` — the build-sidecar
  script has this path hard-coded. Edit `build-sidecar.ps1` if your
  Python is installed elsewhere.
- **Python packages** — `pip install python-pptx pyinstaller`
- **Graphviz** — `winget install Graphviz.Graphviz`. If the installer
  doesn't add it to PATH, add `C:\Program Files\Graphviz\bin` manually.
- **WebView2 Runtime** — preinstalled on Windows 11.

## Full build

```powershell
cd <path-to-mermaidforge>
cd app
npm install      # one-time per fresh clone
cd ..
npm --prefix app run build
```

The `npm run build` chain runs two steps:

1. `build-sidecar.ps1` — copies the Python engines from `engines/` into
   the sidecar working folder, runs PyInstaller to freeze them into a
   standalone exe, verifies the output, and installs it into
   `app/src-tauri/binaries/`.
2. `tauri build` — produces the `.msi` installer.

The installer lands at:

```
app/src-tauri/target/release/bundle/msi/Mermaid Forge_<version>_x64_en-US.msi
```

## Incremental rebuilds during development

If you only changed Rust or JavaScript code (no Python engine changes):

```powershell
npm --prefix app run build:app
```

If you only changed Python engine code:

```powershell
.\build-sidecar.ps1
```

## Project layout

```
mermaidforge/
├── app/                       # Tauri 2.x application
│   ├── src/                   # Frontend: vanilla JS + Mermaid.js
│   ├── src-tauri/             # Rust backend + sidecar config
│   │   ├── python/
│   │   │   ├── cli.py         # Sidecar entrypoint
│   │   │   └── mfengine.spec  # PyInstaller config
│   │   └── binaries/          # Frozen sidecar (gitignored, regenerable)
│   └── package.json
├── engines/                   # Python engines — source of truth
│   ├── mermaid_to_pptx.py
│   ├── mermaid_to_vsdx_generator.py
│   └── template_manual_ref.vsdx
├── design/                    # Icon source files
├── docs/test-corpus/          # Mermaid sample files used as templates
└── build-sidecar.ps1          # Engine → frozen sidecar build script
```

The Python engines are stored canonically in `engines/`. The sidecar
working copies under `app/src-tauri/python/` are regenerated by
`build-sidecar.ps1` on every build and are not tracked in git — never
edit them directly, edit the originals under `engines/`.

## Updating the app icon

The source icon is `design/icon-master.png` (2048×2048). After changing
it, regenerate the platform icon set:

```powershell
cd app
npx tauri icon ..\design\icon-master.png
```

This populates `app/src-tauri/icons/` with every sized variant Tauri
needs (32x32, 128x128, 128x128@2x, icon.ico, icon.icns, plus various
MS Store / iOS / Android variants — only the Windows icons ship in the
MSI; the mobile assets sit on disk unused).

The 24×24 toolbar icon at `app/src/icon-24.png` is generated separately
because `npx tauri icon` doesn't emit a 24×24. Regenerate with:

```powershell
Add-Type -AssemblyName System.Drawing
$src = "design\icon-master.png"
$dst = "app\src\icon-24.png"
$srcImg = [System.Drawing.Image]::FromFile((Resolve-Path $src).Path)
$dstImg = New-Object System.Drawing.Bitmap(24, 24)
$g = [System.Drawing.Graphics]::FromImage($dstImg)
$g.InterpolationMode = [System.Drawing.Drawing2D.InterpolationMode]::HighQualityBicubic
$g.SmoothingMode = [System.Drawing.Drawing2D.SmoothingMode]::HighQuality
$g.PixelOffsetMode = [System.Drawing.Drawing2D.PixelOffsetMode]::HighQuality
$g.DrawImage($srcImg, 0, 0, 24, 24)
$dstImg.Save((Resolve-Path $dst).Path, [System.Drawing.Imaging.ImageFormat]::Png)
$g.Dispose(); $dstImg.Dispose(); $srcImg.Dispose()
```

Rebuild with `npm --prefix app run build:app` to pick up icon changes
in the next `.msi`.
